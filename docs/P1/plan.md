# P1 阶段计划书（变现能力试点）

## 1. 阶段目标
- 在增长闭环基础上验证变现可行性，建立第一条可持续收入路径。
- 采用“最小可用变现模型”，优先验证转化，不追求复杂产品形态。

## 2. 业务范围
- 会员/付费内容：先支持单篇或专栏访问控制。
- 打赏：文章级打赏入口与记录。
- 广告：固定广告位配置与投放统计。

## 3. 关键交付
- 内容可配置免费/付费可见策略。
- 用户可完成支付或打赏并产生可核对记录。
- 广告位可管理并支持点击统计。
- 后台具备变现看板（订单、收入、转化、广告点击）。

## 4. 指标与验收
- 付费转化率、打赏转化率、广告点击率可持续观测。
- 连续 2 周有稳定收入事件（非 0）。
- 关键支付链路成功率达到可运营水平。

## 5. 风险与控制
- 风险：过早投入复杂支付和结算设计拖慢进度。
- 控制：先最小化接入，优先打通交易闭环与对账闭环。

## 6. 阶段完成定义
- 变现能力形成可复盘的数据闭环。
- 具备继续放大流量和优化转化的基础设施。
